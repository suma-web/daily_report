- あまり意識していなかった部分
    - イベントハンドラに渡す関数は、渡すべきなのであって、呼び出すべきではない。

|関数を渡す（正しい）|関数を呼び出す（間違い）|
|---|---|
|`<button onClick={() => alert('...')}>	`|`<button onClick={alert('...')}>`|
- 慣習として、イベントハンドラのプロップは on で始まり、次に大文字の文字が続くようにする。
- イベントが親コンポーネントに伝わらないようにしたい場合、e.stopPropagation() を呼び出す。
```
function Button({ onClick, children }) {
  return (
    <button onClick={e => {
      e.stopPropagation();  //buttonの機能として定義。
      onClick();
    }}>
      {children}
    </button>
  );
}

export default function Toolbar() {
  return (
    <div className="Toolbar" onClick={() => {
      alert('You clicked on the toolbar!');
    }}>
      <Button onClick={() => alert('Playing!')}>
        Play Movie
      </Button>
      <Button onClick={() => alert('Uploading!')}>
        Upload Image
      </Button>
    </div>
  );
}
```
```
e.stopPropagation( ) は、ツリーの上側にあるタグにアタッチされたイベントハンドラが発火しないようにする。
e.preventDefault( ) は、イベントがブラウザデフォルトの動作を持っていた場合に、それを抑制。
```
- `useState` を呼び出すことはコンポーネントに何かを覚えさせるよう React に指示を出すということ。以下でインポートする。
```
import { useState } from 'react';
```
```
const [index, setIndex] = useState(0);
※ index は state 変数、setIndex はセッタ関数

1. useState に index の初期値として 0 を渡したので、[0, setIndex] を返す。React は 0 が最新の state 値であることを覚える。
2. ユーザがボタンをクリックすると、setIndex(index + 1) が呼び出される。現在 index は 0 なので、setIndex(1) になる。これにより、React は index が 1 になったことを覚え、再レンダーがトリガされる。
3. コンポーネントの 2 回目のレンダー。React は再び useState(0) というコードに出会うが、React は index を 1 にセットしたことを覚えているので、代わりに [1, setIndex] を返す。
4. 以降も続く。
```
- state は独立しておりプライベート。同じコンポーネントを 2 回レンダーした場合、それぞれのコピーは完全に独立した state を有することになる。
- state 変数は、コンポーネントの再レンダー間で情報を保持するためにのみ必要なもの。単一のイベントハンドラ内では、通常の変数で十分。通常の変数でうまく動作する場合は、state 変数を追加しない。
