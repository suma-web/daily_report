# わかったこと
- `n => n + 1` は更新用関数
    - state 変数を複数回更新する場合、`setNumber(n => n + 1) `のような state を計算する関数を渡すことが可能。
- React のすべての state はイミュータブルとして扱う。
- 直接、`person.firstName = "Alice";`のように変更しない。`setPerson({...})`する。
- Immer で簡潔な更新ロジックを書く
    - `npm install use-immer` を実行し、Immer を依存ライブラリとして追加する
    - `import { useState } from 'react'` を `import { useImmer } from 'use-immer'` に置き換える
- state 内の配列では、配列を書き換えずに更新する
    - 〈追加〉`concat`, `[...arr]`
    - 〈削除〉`filter` , `slice` 
    - 〈要素置換〉`map`
    - 〈ソート〉先に配列をコピー `（例） const nextList = [...list];`
- 深くネストされた state の更新が複雑な場合は、フラット化を試す。
