# わかったこと
- 関数式と関数宣言は異なる
- グローバルスコープ
    - windowオブジェクト＝グローバルスコープ（ただし一部ではスコープチェーンという仕組みがある）
- ブロックスコープとは{}内で宣言されているもの（ifやforなどを使う）
    - var や関数は無視される
- `~Factory`と変数名でつける場合は「何かを生成する関数」の時についていることが多い。
- `return`は関数名を返すことも可能。
- クロージャーを利用。プログラムの他の部分（外側のコード）からは直接触れさせたくないデータを隠しつつ、そのデータを操作するための必要な機能だけを作れる。
- 関数の特性上、名前をいちいちつける必要があるので、即時関数は一度だけしか使用しないものに関して有効。
```
function a(){
    console.log('called');
}

a();

(function() {
    console.log('called');
})();

(a)();と同じ。
```

- 即時関数と`return`を使った演習で印象的だった問題
```
function calcFactory(a){
    return {
        plus: function (num){
            const newVal = a + num;
            console.log(newVal);
            a = newVal;
        },
        minus: function(num){
            const newVal = a - num;
            console.log(newVal);
            a = newVal; 
        },
        multiply: function(num){
            const newVal = a * num;
            console.log(newVal);
            a = newVal;
        },
        divide: function(num){
            const newVal = a / num;
            console.log(newVal);
            a = newVal;
        }
    }
}
```
