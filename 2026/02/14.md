# わかったこと
- 条件 (三項) 演算子 (`? :`)
```
return (
  <li className="item">
    {isPacked ? name + ' ✅' : name}
  </li>
);
```
- `$$`は左辺がtruethyなら`✅`を返す。JSXは左辺に数値を置かない。
    - 例：`{isPacked && '✅'}`

- 変数を設け、柔軟性があるコードに。
```
if (isPacked) {
  itemContent = name + " ✅";
}

<li className="item">
  {itemContent}
</li>
```
- `map( )` 内で直接 JSX 要素を使用する場合、必ず key が必要。配列内の他のアイテムと区別できるようにするための一意な文字列ないし数値。
    - `<li key={person.id}>...</li>`
- `import { Fragment } from 'react'`はReactが用意しているコンポーネント。余計なDOMを作らないためのラッパー。
- コンポーネントは純粋である必要がある。
    - コンポーネントは自分の仕事に集中する。レンダー前に存在していたオブジェクトや変数を書き換えない。
    - 入力が同じなら出力も同じ。同じ入力に対しては、常に同じ JSX を返すようにする
- レンダー中にその場で作成した変数やオブジェクトであれば、書き換えることは問題ない。
