# わかったこと
- ReactをViteで起動するコマンド
```
npm create vite@latest
```
どんなReact環境を自動生成するか設定
```
（Project name: 任意
Framework: Reactなど
Variant: JavaScriptなど）
```
起動サーバー
```
http://localhost:5173/
```
再度サーバー立ち上げ
```
npm run dev
```

- 関数は名前は大文字から始める必要がある
- すべてのコンポーネントをトップレベルで定義する。（ネストさせない）
- すべてキャメルケース
    - 名前にハイフンを含めたり class のような予約語を使ったりすることはできない。
- すべてのタグを閉じる（明示的に）
    - <img> のような自動で閉じるタグは <img /> のようになる。<li>oranges のような囲みタグは <li>oranges</li> と書く。
```
<b>And <i>pictures</b></i> of scientists!   //iタグの途中に違うタグがあってはダメ
<b>And <i>pictures</i></b> of scientists!   //正解⚪︎
```
- class は予約語なので、React では `className`
- JSX に文字列の属性を渡したい場合、それをシングルクオートかダブルクオートで囲む
- src や alt のテキストを動的に指定したい場合はどうすればいいのでしょう？ " と " を { と } に置き換えることで、JavaScript の値を使うことができる。
- オブジェクトを別の波括弧のペアでラップ。
```
person={{ name: "Hedy Lamarr", inventions: 5 }} 
```
- 画像 URL を指定するところで、ベース URL（常に 'https://i.imgur.com/'）、imageId ('7vQD0fP')、imageSize ('s')、ファイル拡張子（常に '.jpg'）の 4 つの値を結合できる
```
src={baseUrl + person.imageId + person.imageSize + '.jpg'}
```
- 親コンポーネントは子コンポーネントに props を渡すことで情報を伝えることができる
- 子コンポーネントから props を読み出す 
    - これらの props を読み出すには、functionの直後の ({ と }) とする
