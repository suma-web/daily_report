# わかったこと
- グローバルスコープで定義した変数はグローバル変数と呼ばれ、グローバル変数はあらゆるスコープから参照できる変数となる。
- 変数に格納される時や新たに格納されるものに関してメモリが使用される
- thisはキーワードであるため、ユーザーは`this`という名前の変数を定義できない
- Arrow Functionで定義した関数ではcall、apply、bindを使ったthisの指定は単に無視される。 Arrow Functionはthisを持てないから。

- 小クラスと親クラスのルール
```
class Parent {
    constructor() {
        this.name = "Parent";
    }
}
class Child extends Parent {
    constructor() {
        // 子クラスでは`super()`を`this`に触る前に呼び出さなければならない
        super();
        // 子クラスのコンストラクタ処理
        // 親クラスで書き込まれた`name`は上書きされる
        this.name = "Child";
    }
}
const parent = new Parent();
console.log(parent.name); // => "Parent"
const child = new Child();
console.log(child.name); // => "Child"
```
