# わかったこと
**startprojectでできること**
- setting.py
    - BASE.DIR 作ったファイルを保存する基準となる場所を指定
    - Secret KEY Djangoの一つひとつのプロジェクトで作られる秘密鍵。開発ではあまり意識することはない。
    - DEBUG = True はエラーを詳細に書いてもらう設定（開発時に使用）、DEBUG = Falseはウェブサイト上に公開したときに設定。詳細を見られないようにするためにする。
    - ALLOWED HOSTSはDEBUG = Falseの場合に指定したサーバーからだけのアクセスにするように設定

- Urls.py
    - urlpatterns のpathと右側が実際に受け取るリクエストのこと（例：path('admin/', admin.site.urls)　これは管理画面が出る）
        - Python3 manage.py migrate
        - Python3 manage.py runserver
            - http://127.0.0.1:8000/path名 で開く（今回はhttp://127.0.0.1:8000/admin で管理画面になる）
                - Path名はadminでなくても名前の変更はできる

- Views.py
    - __pycache の中にviews.py を作成（touch views.py）。デフォルトでは用意されていない。
    - Views.pyは関数やモジュールなどの2種類で呼び出すことができる
