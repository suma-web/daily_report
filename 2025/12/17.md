# わかったこと
- Class based view
    - あまり中身の指示をしなくてもWebサイトを作れるようになっている
- Function based view
    - 一個一個細い指示をして行かなければならないけども調整しやすい
- urls.pyでは、どのHTMLファイルを読み込むのかの指示ができる（htmlファイル作成時は、manage.pyの階層にいることを確認。ディレクトリ作成後、その配下にhtmlファイルを作成）
- http://127.0.0.1:8000/path名  を localhost:8000/と打ち込んでも同じ意味になる
- アプリ作成
    - `python3 manage.py startapp アプリ名`（manage.pyの階層にいることを確認）
        - アプリを作るための一式のツールが作成できる
- Projectディレクトリ傘下のUrls.pyについて
    - urlpatterns の設定で、path(‘’,include(‘todo.urls’) と’’の中が空欄だと、自動的にurls.pyファイルを呼び出す
