# わかったこと
- DCL（データ制御言語）
    - 誰に、どのようなデータ操作やテーブル操作を許すかといった権限を設定
        - GRANT文（権限を付与）【GRANT 権限名 TO ユーザ名】
        - REVOKE文（権限を剥奪）【REVOKE 権限名 FROM ユーザ名】
- 同じ名前でデータベースを作ることはできない
- `ALTER TABLE テーブル名 DROP カラム名/ADD カラム名 データ型` （テーブルの定義の変更）
- テーブルの制約
    - NOT NULL制約では, DEFAULT指定と組み合わせるのが一般的
    - 主キー制約　そのテーブで管理している一意に識別する列であるという意味
	    - 複合キーの場合, `PRIMARY KEY(ID, 名前)`
- インデックスを作成し、より高速な索引を実現する
    - なんでも作ればいいわけではない。where句やjoin句、orderby句において効果が現れやすい。一方で, EXPLAINで実行計画を考えるのも一つの方法
- 連番を作る方法はDBMSによって異なる
	- PostgreSQL -> `SELECT NEXTVAL(sequence名)`　シーケンスの作成をする必要があるがまずは扱い方
- 概念設計→論理設計→物理設計
- 非正規化も便利だが、最終手段の方法
